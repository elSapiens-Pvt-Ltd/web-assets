Conventions

Schema Conventions

  • UUID primary keys — gen_random_uuid() for all IDs, client-generated for offline-created records
  • BIGINT paisa — all monetary values in paisa (₹1 = 100). Zero floating-point usage
  • Tenant isolation — (organization_id, workspace_id) on every row. Always included in WHERE clauses
  • Soft delete — deleted_at TIMESTAMPTZ on master data tables. Partial indexes exclude deleted rows
  • Offline idempotency — offline_id UUID UNIQUE NULLS NOT DISTINCT prevents duplicate processing
  • Immutable ledger — ledger_entries and wallet_transactions are append-only. No UPDATE or DELETE
  • JSONB for flexibility — Rule conditions and reward configs use JSONB for schema evolution without migrations
  • Partial indexes — Active records indexed with WHERE deleted_at IS NULL or WHERE status = 'active'
