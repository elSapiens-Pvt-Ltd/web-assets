Auth

Authentication & Authorization

elLoyalty never manages its own user credentials. All authentication flows through elAuth JWTs with workspace-scoped claims.

┌────────────┐     JWT       ┌──────────────────────────────────────────┐
│  POS / Admin│──────────────▶│  elLoyalty Middleware                    │
│  Client     │              │                                          │
└────────────┘              │  1. Validate JWT signature (HS256)       │
                             │  2. Check expiration (exp claim)         │
                             │  3. Extract workspace_id, org_id         │

// Middleware: enforce role-based access func RequireRole(roles ...string) func(http.Handler) http.Handler { return func(next http.Handler) http.Handler { return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) { tc := TenantContextFromCtx(r.Context()) for _, required := range roles { if tc.Role == required { next.ServeHTTP(w, r) return } } http.Error(w, "forbidden", http.StatusForbidden) }) } } // Usage: r.With(RequireRole("org_admin", "workspace_admin")).Post("/rules", h.CreateRule) r.With(RequireRole("terminal", "store_admin")).Post("/evaluate", h.EvaluateCart)
Tenant

Tenant Isolation

Every database query is scoped to (workspace_id, organization_id) extracted from JWT. No cross-tenant data leakage is possible.

// Every repository method enforces tenant scope func (r *RuleRepo) ListActive(ctx context.Context, tc TenantContext) ([]BenefitRule, error) { rows, err := r.db.Query(ctx, `SELECT id, name, conditions, reward, priority, stackable, is_active FROM benefit_rules WHERE workspace_id = $1 AND organization_id = $2 AND is_active = true AND (valid_from IS NULL OR valid_from <= NOW()) AND (valid_to IS NULL OR valid_to >= NOW()) ORDER BY priority DESC`, tc.WorkspaceID, // $1 — always from JWT, never from request body tc.OrganizationID, // $2 — always from JWT, never from request body ) // ... scan rows }

Never Trust Client IDs

Workspace and organization IDs come exclusively from the JWT. Request bodies may include customer_id and sale_id, but tenant scope is always server-enforced.

Row-Level Isolation

Every table has (workspace_id, organization_id) columns. Composite indexes ensure queries only scan tenant-relevant rows.

Fraud

Fraud Prevention

Threat: Coin Farming

A bad actor creates fake sales or inflates sale amounts to earn excessive loyalty coins, then redeems them.

Mitigation: Velocity Checks

Track coins earned per customer per day/week. If threshold exceeded, flag for review and stop auto-crediting. Configurable per organization.

Threat: Coupon Abuse

A customer reuses single-use coupons by exploiting offline mode or by sharing codes across terminals before sync.

Mitigation: Post-Sync Validation

Offline coupon usage is provisional. On sync, server validates usage count. If exceeded, the benefit is reversed and flagged. Customer has a debt entry.

Threat: Cashier Manipulation

A cashier manually credits coins to a friend's loyalty account or applies benefits to a sale that shouldn't qualify.

Mitigation: Audit Trail + Manual Credit Limits

Manual credits require store_admin role and are logged with actor_id. Daily manual credit limit is configurable. All benefit applications are tied to a sale_id.

Velocity Rules

type VelocityConfig struct { MaxCoinsPerDay int64 `json:"max_coins_per_day"` // default: 1000 MaxRedemptionsPerDay int `json:"max_redemptions_per_day"` // default: 10 MaxCashbackPerWeek int64 `json:"max_cashback_per_week"` // paisa, default: 500000 (₹5000) MaxManualCreditPerDay int64 `json:"max_manual_credit_per_day"` // paisa, default: 100000 (₹1000) CooldownAfterFlag time.Duration `json:"cooldown"` // default: 24h } func (s *FraudService) CheckVelocity(ctx context.Context, customerID uuid.UUID, action string) error { key := fmt.Sprintf("velocity:%s:%s:%s", customerID, action, today()) count, _ := s.redis.Incr(ctx, key).Result() s.redis.Expire(ctx, key, 24*time.Hour) limit := s.config.LimitFor(action) if count > int64(limit) { s.flagCustomer(ctx, customerID, action, count) return ErrVelocityExceeded } return nil }

// Redis-backed sliding window rate limiter func RateLimit(redis *redis.Client, limit int, window time.Duration) func(http.Handler) http.Handler { return func(next http.Handler) http.Handler { return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) { tc := TenantContextFromCtx(r.Context()) key := fmt.Sprintf("rl:%s:%s", r.URL.Path, tc.TerminalID) count, _ := redis.Incr(r.Context(), key).Result() if count == 1 { redis.Expire(r.Context(), key, window) } w.Header().Set("X-RateLimit-Remaining", fmt.Sprintf("%d", max(0, int64(limit)-count))) if count > int64(limit) { http.Error(w, "rate limit exceeded", http.StatusTooManyRequests) return } next.ServeHTTP(w, r) }) } }
