elAuth

elAuth Integration

Authentication, authorization, and multi-tenancy context.

JWT Validation Middleware

pkg/middleware/auth.go

Every HTTP and gRPC request to elLoyalty carries a JWT issued by elAuth. The middleware extracts tenant context (workspace_id, organization_id) and user identity from JWT claims.

// JWT claims extracted by elAuth middleware type TenantContext struct { WorkspaceID uuid.UUID `json:"workspace_id"` OrganizationID uuid.UUID `json:"organization_id"` PersonID string `json:"person_id"` // MongoDB ObjectID from elAccounts StoreID *uuid.UUID `json:"store_id"` // nil for admin, set for terminal } // gRPC: Fetch tenant config for loyalty behavior func (c *AuthClient) GetTenantConfig(ctx context.Context, domain string) (*TenantConfig, error) { resp, err := c.configClient.GetConfig(ctx, &configpb.GetConfigRequest{ Domain: domain, }) if err != nil { return nil, fmt.Errorf("elAuth config: %w", err) } return mapTenantConfig(resp), nil }

elPOS

elPOS Integration

The primary consumer of loyalty benefits. elPOS calls elLoyalty during checkout and publishes sale events.

gRPC: LoyaltyService (elLoyalty exposes)

proto/loyalty/loyalty.proto — port 6010
// Proto service called by elPOS service LoyaltyService { // Phase 1: Evaluate available benefits for a cart rpc EvaluateCart (EvaluateCartRequest) returns (EvaluateCartResponse); // Phase 2: Apply selected benefits to finalize sale rpc ApplyBenefits (ApplyBenefitsRequest) returns (ApplyBenefitsResponse); // Phase 3: Credit earned benefits after sale rpc CreditEarnings (CreditEarningsRequest) returns (CreditEarningsResponse); // Customer lookup for POS display rpc GetCustomerWallet (GetCustomerWalletRequest) returns (CustomerWalletResponse); // Offline sync endpoints rpc SyncPush (SyncPushRequest) returns (SyncPushResponse); rpc SyncPull (SyncPullRequest) returns (SyncPullResponse); }

Resilience

Circuit Breaker Pattern

All gRPC clients use sony/gobreaker for graceful degradation when downstream services are unavailable.

import "github.com/sony/gobreaker/v2" func NewCatalogClient(conn *grpc.ClientConn) *CatalogClient { cb := gobreaker.NewCircuitBreaker[*catalogpb.GetProductsResponse](gobreaker.Settings{ Name: "elCatalog", MaxRequests: 3, // half-open: allow 3 test requests Interval: 30 * time.Second, Timeout: 10 * time.Second, ReadyToTrip: func(counts gobreaker.Counts) bool { return counts.ConsecutiveFailures > 5 }, }) return &CatalogClient{client: catalogpb.NewProductServiceClient(conn), cb: cb} }

elCatalog Down

Skip category/brand rules. Evaluate only cart-total and customer-tier rules. Coins still earned at base rate.

elbooks Down

Queue journal entries in Redis. A background worker retries when elbooks is reachable. No customer-facing impact.

elmessagehub Down

Notifications are fire-and-forget via Redis pub/sub. Missed notifications are logged but do not block operations.

elAuth Down

JWT validation is local (shared secret / JWKS cache). Config refresh fails gracefully to last-known config.
