How Feature Gating Works

When a business buys an elPOS plan (Starter, Pro, Enterprise), the plan includes features with limits. elPOS checks these at runtime before allowing any action:

CheckEntitlement(workspace_id, "mode_kot")
→ { entitled: false }
→ UI shows: "Restaurant mode requires elPOS Pro. Upgrade to unlock."

CheckEntitlement(workspace_id, "max_terminals")
→ { entitled: true, used: 2, limit: 3, remaining: 1 }
→ Allow terminal registration

Feature types: quota has a numeric limit (can be unlimited). boolean is simply on or off.

1. Infrastructure

4 features
Feature KeyTypeDescription
max_terminalsquotaPOS terminals/registers — each physical device or browser tab running elPOS counts as one terminal. A laundry shop with 1 counter needs 1. A supermarket with 5 billing counters needs 5.
max_storesquotaStore locations — each physical branch/outlet. A single-shop business has 1. A chain with 3 branches has 3. Each store can have multiple terminals.
max_staff_accountsquotaStaff logins — how many people can operate the POS. Includes cashiers, managers, and owners. Each person gets their own login for accountability.
max_concurrent_sessionsquotaRegisters open at the same time — a store with 3 terminals but only 2 staff working simultaneously needs 2 concurrent sessions.
