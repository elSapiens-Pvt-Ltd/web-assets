GST Calculation Engine

internal/gst — No API endpoints (internal service)
Components
gst/calculator.go
CalculateGST(amount, rate, sellerState, buyerState, inclusive) → GSTResult with CGST/SGST/IGST/cess breakdown.
NEW
grpc/geolocations_client.go
GetState() for tax codes. Redis 24hr cache. Circuit breaker with hardcoded fallback.
NEW
Calculation Flow
Sale item added → Get HSN/SAC code from elCatalogs → Get GST rate → Get seller state from Inventory (store address) → Get buyer state from customer (or default to seller state for B2C) → Same state? YES → CGST = rate/2, SGST = rate/2 (e.g., 9% + 9% = 18%) NO → IGST = full rate (e.g., 18%) → If tax-inclusive: reverse calculate (base = amount / (1 + rate/100)) → Return: taxable, CGST amt, SGST amt, IGST amt, cess

Key User Flows

Flow 1: First Time User

User opens pos.elsapiens.com → JWT cookie present? → NO → Redirect to auth.elsapiens.com → Login → Back with JWT → JWT valid → elPOS loads → GET /stores → Empty → Show Setup Wizard → Step 1: Create Store → POST /stores → Inventory: CreateStore + CreateWarehouse → Step 2: Register Terminal → POST /terminals → INSERT terminals → Step 3: Open Register → POST /sessions/open → INSERT register_sessions → POS screen loads — ready for first sale

Flow 2: Daily Sale (Returning User)

Staff opens tablet → JWT cookie valid → elPOS loads → GET /sessions/current?terminal_id=T1 → No session → Show "Open Register" → Enter ₹5,000 float → POST /sessions/open → POS ready Customer arrives → Type "rice" → GET /products?search=rice → elCatalogs: SearchItems → Results shown → Tap "Rice 5kg" → POST /sales/{id}/items → GST calculated → Cart updates → Tap "Dal 1kg" × 2 → POST /sales/{id}/items → Cart: ₹590 + ₹29.50 GST = ₹620 → Tap [Pay] → [Cash] → Customer gives ₹700 → Change ₹80 → POST /payments → Sale completed → Bill# T1-INV-2526-0001 → Inventory: DeductStock (async) → elAccounts: PostJournal (async) → Receipt prints → Cart resets → Ready for next customer

Flow 3: Laundry Service Order

Customer drops off clothes → Search customer phone → GET /customers?phone=9876 → Found: Meera → Tap garments: 2×Shirt, 1×Pants, 1×Blazer → Select services: Shirt→Wet Clean(₹40), Blazer→Dry Clean(₹350) → Photograph each garment (camera pre-initialized) → Tag blazer: [Stain] [Minor Fade] → Set delivery: 28-Mar 6PM → POST /orders → Order #T1-SO-2526-0001 created (status: received) → Garment tags print → Receipt given to Meera 2 days later — processing → Staff opens Orders page → Taps "Start Processing" → PATCH /orders/{id}/status → in_progress → elmessagehub: "Your order is being processed" → Staff marks each item done → All done → Taps "Mark Ready" → PATCH /orders/{id}/status → ready → elmessagehub: "Your order is READY for pickup!" Meera picks up → Scan order# → GET /orders/{id} → Status: READY → POST /orders/{id}/deliver → status → delivered → POST /orders/{id}/invoice → Sale created: ₹549 with GST → POST /payments (UPI) → QR shown → Meera scans → Paid → Receipt prints → Done

Flow 4: End of Day Settlement

End of shift — Priya closes register → Tap [Settle] or press F12 → System calculates expected: opening float + cash sales - change given → Priya counts cash by denomination: ₹2000 × 2 = ₹4,000 ₹500 × 28 = ₹14,000 ₹100 × 22 = ₹2,200 ... total: ₹21,525 → POST /settlements (actual=₹21,525, expected=₹21,525, variance=₹0) → Submitted for manager approval Manager Rahul reviews → PATCH /settlements/{id}/approve → Session closed → elmessagehub: Daily summary to Rahul's WhatsApp "Sales: ₹44,905 | 23 txns | Cash: ₹21,525 | Card: ₹15,200 | UPI: ₹11,205 | Variance: ₹0 ✓"
